6  从单次计算到长期运行任务

6.1 本章要回答的问题

前面讲的是一次计算如何发生。但平台工程真正面对的不是一次计算，而是长期运行的任务和服务。

单次 forward 只是局部动作；训练任务和推理服务是由大量计算、状态、资源和控制逻辑组成的生命周期系统。

6.2 单次计算、训练任务和推理服务

层级 例子 关注点
单次 operator 一次矩阵乘法、一次 attention shape、dtype、显存访问、底层实现
一个 step 一个训练 batch 的 forward/backward/update step time、activation、gradient、optimizer
一个 job 一次完整训练或微调 数据、checkpoint、恢复、指标、资源成本
一个 request 一次推理请求 TTFT、TPOT、队列、KV Cache
一个 service 长期在线模型服务 并发、SLA、扩缩容、灰度、故障隔离

这张表说明为什么本书要循序渐进：如果不理解 operator，就很难理解 step；如果不理解 step，就很难理解训练 job；如果不理解 request，就很难理解推理 service。

6.3 生命周期里最容易被忽略的状态

训练和推理都不是无状态过程。

训练状态包括：

  • 当前 parameters。
  • optimizer state。
  • 当前 step、epoch 和随机数状态。
  • checkpoint 文件。
  • 数据读取位置和实验记录。

推理状态包括：

  • 已加载的 weights。
  • 当前请求队列。
  • KV Cache。
  • 模型版本和路由状态。
  • 副本健康状态。

平台要做的不是“把容器启动起来”，而是管理这些状态如何创建、使用、保存、迁移和恢复。

6.4 从一次计算到平台对象

现在可以把第一篇的基础概念映射成后续会出现的平台对象：

基础概念 后续会变成什么平台问题
operator GPU kernel、算子融合、Profiler、性能瓶颈
step 训练吞吐、checkpoint 间隔、失败恢复粒度
job 队列、优先级、Gang Scheduling、抢占策略
request 路由、限流、batching、超时、SLA
service 副本、灰度、回滚、扩缩容、健康检查
state checkpoint、KV Cache、模型版本、实验记录

这张表是全书后续的导航。Kubernetes 不是凭空引入的，它是在这些生命周期对象变复杂之后出现的承载层：Job 承载训练任务，Deployment/Service 承载在线推理，CRD/Operator 承载领域语义，调度器承载资源决策。

6.5 判断一个 AI workload 的五个问题

以后看到任何 AI 任务，先问五个问题：

  1. 它是训练、推理、数据处理，还是混合任务？
  2. 它的核心状态是什么：parameters、optimizer state、checkpoint、KV Cache 还是数据游标？
  3. 它的失败代价是什么：重跑一个 step、重跑整个 job，还是丢失一个在线请求？
  4. 它的性能目标是什么：step time、tokens/sec、TTFT、TPOT、P99 还是单位成本？
  5. 它的资源瓶颈最可能在哪里：数据、计算、显存、通信还是控制流？

这五个问题比“它需要几个 Pod”更靠近本质。只有先回答这些问题，后面谈 Kubernetes 配置才有意义。

第一篇如何连接后续章节

每个基础概念都会在后文变成一个具体系统问题。

这也是本书的写法原则：前面出现的概念，后面必须变成可分析、可优化、可部署的系统问题。比如 activation 不是只在第一篇出现一次的名词，它会在第二篇变成 activation checkpointing 和训练显存优化；KV Cache 不是推理服务里的黑话，它会在第三篇变成上下文长度、并发、continuous batching 和副本容量；五条资源流也会在 Kubernetes 篇变成调度、观测和资源治理方法。

6.7 本章小结

第一篇到这里完成基础闭环：

  • AI 系统主线是 data -> tensor -> parameters -> compute -> result。
  • tensor shape 决定计算量、显存和并行度。
  • 模型运行时要看 parameters、activation、gradient、KV Cache 和 buffer。
  • 训练和推理是两种不同生命周期系统。
  • AI workload 可以用五条资源流分析。

第二篇开始，我们会进入训练流程：数据如何进入系统，forward/backward 如何产生 gradient，optimizer 如何更新 parameters，checkpoint 如何让长任务可恢复。
