23  HBM、显存带宽与显存容量

23.1 本章要回答的问题

GPU 快，不只是因为算得快，还因为靠近计算单元的显存带宽高。但显存同时也是最硬的资源约束。

显存容量决定能不能跑，显存带宽决定很多 operator 跑得快不快。

Bandwidth：数据搬运速度

L2/SM更近更快
Host远，搬运贵

OOM 是容量问题；GPU 忙但吞吐低，可能是带宽或数据复用问题。

23.2 容量账本

训练显存通常包括：

内容 为什么占显存
weights forward/backward 都要使用
activation backward 要依赖 forward 的中间结果
gradient optimizer 更新 parameters 所需
optimizer state Adam 等 optimizer 保存额外状态
communication buffer 分布式训练同步或切分状态

推理显存通常包括：

内容 为什么占显存
weights 模型常驻显存
KV Cache LLM decode 复用历史 key/value
runtime buffer operator 临时空间、请求状态
batching 余量 并发和上下文长度的安全空间

这解释了为什么“模型 weights 能放下”不等于训练能跑，也不等于推理能承载高并发。

23.3 容量估算要留余量

显存估算不能只加静态对象。真实运行还要留 runtime buffer、通信 buffer、临时 workspace 和碎片余量。

场景 容易漏算什么
训练 optimizer state、activation 峰值、梯度累积、通信 buffer
LLM 推理 KV Cache、长尾上下文、continuous batching、prefix cache
多卡推理 tensor parallel 通信 buffer、跨卡激活、临时 workspace
编译/优化 runtime 图编译、kernel workspace、缓存池

所以“理论上 79GB，卡有 80GB”通常是不安全的。生产系统要留显存余量，否则稍微遇到长上下文、batch 波动或 runtime workspace，就可能 OOM。

23.4 带宽与 arithmetic intensity

NVIDIA 性能资料使用 arithmetic intensity 判断一个操作更可能受算力限制还是内存限制：每访问一个字节能做多少计算，决定它更接近 math-limited 还是 memory-limited (《GPU Performance Background User’s Guide》, 不详)。

直白地说：

  • 矩阵乘法通常数据复用高，更可能吃 Tensor Core 和算力。
  • elementwise、reshape、部分 normalization 可能频繁搬数据，更容易受带宽限制。
  • 如果数据来自 CPU、存储或网络，瓶颈可能根本不在 HBM。

23.5 memory-bound 和 compute-bound 怎么判断

可以用一个简单问题判断：GPU 时间主要花在计算，还是花在搬数据？

类型 典型表现 优化方向
compute-bound Tensor Core/SM 忙，HBM 不是瓶颈 混合精度、shape 对齐、更强 GPU
memory-bound HBM 带宽高，SM 等数据 算子融合、减少中间 tensor、提高数据复用
launch-bound 小 kernel 很多，CPU 发起频繁 编译优化、算子融合、增大 batch
input-bound GPU 等 CPU/存储 DataLoader、缓存、tokenizer 并行、预取

NVIDIA 的性能资料用 arithmetic intensity 来描述这个边界 (《GPU Performance Background User’s Guide》, 不详)。平台工程不必手算每个 operator，但要知道：矩阵乘法和 elementwise 的瓶颈很可能不同，不能用同一种优化手段。

23.6 显存碎片与缓存池

深度学习框架通常会使用显存缓存池，避免频繁向驱动申请和释放显存。这会导致一个现象：监控里看到的 reserved memory 很高，不等于 tensor 真实占用同样高。

推理 runtime 还会管理 KV Cache block。如果长短请求混合，cache 分配和回收不均匀，就可能出现碎片：总空闲显存看似够，但找不到合适连续空间或可用 block。vLLM 这类 runtime 围绕 KV Cache block 管理做优化，就是为了解决这类问题。

23.7 本章小结

  • 显存容量回答“能不能放下”。
  • 显存带宽回答“数据能不能及时送到计算单元”。
  • 训练和推理的显存账本不同，不能用同一套估算方式。
  • 下一章讲互联：当一张 GPU 不够时，数据如何在 GPU、节点和网络之间移动。
《GPU Performance Background User’s Guide》. 不详. https://docs.nvidia.com/deeplearning/performance/dl-performance-gpu-background/index.html.
