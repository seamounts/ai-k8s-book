24  PCIe、NVLink、RDMA 与 InfiniBand

24.1 本章要回答的问题

多 GPU 不只是“GPU 数量更多”。只要跨 GPU 或跨节点，就会出现通信成本。

分布式 AI 的扩展效率，取决于计算节省是否大于通信和同步成本。

Excalidraw Style

GPU 互联：8 张卡不等于一块大 GPU

手绘风格源文件：fig/excalidraw/interconnect-map.excalidraw

CPU-GPUPCIe / NUMA数据加载、模型加载、host-device copy。
GPU-GPUNVLink / NVSwitch单机多卡训练和模型并行。
Node-NodeRDMA / IB多节点 AllReduce、参数切分、跨节点推理。
Scheduler拓扑感知知道 GPU 数量，也要知道它们怎么连。

24.2 三类数据移动

路径 例子 影响
CPU 与 GPU host memory 到 HBM 数据加载、模型加载、同步
单机多 GPU PCIe、NVLink、NVSwitch DDP、tensor parallel、pipeline parallel
多节点 RDMA、InfiniBand、以太网 AllReduce、参数切分、分布式推理

同样是 8 张 GPU，单机 8 卡和跨 4 个节点各 2 卡，通信路径完全不同。

24.3 通信模式和训练策略的关系

并行策略 主要通信 拓扑敏感度
Data Parallel / DDP gradient AllReduce 中到高，取决于模型和 step
FSDP / ZeRO AllGather、ReduceScatter 高，参数和 optimizer state 频繁分片
Tensor Parallel 层内 AllReduce/AllGather 很高，最好在高速 GPU 互联内
Pipeline Parallel stage 间 activation 传递 取决于切分和 micro-batch
多节点推理 KV/activation 或请求路由 取决于模型切分和架构

所以“任务需要 8 张 GPU”是不完整描述。还要问：这 8 张 GPU 是用于数据并行、张量并行还是流水线并行？通信发生在 step 末尾，还是每一层都发生？答案会决定它适合单机、跨节点，还是必须要高速互联。

24.4 NCCL 解决什么

NCCL 是 NVIDIA 面向多 GPU、多节点通信的集合通信库，支持 AllReduce、Broadcast、Reduce、AllGather、ReduceScatter 等通信模式 (《NVIDIA NCCL User Guide》, 不详)。

训练里的 DDP 常用 AllReduce 同步 gradient；FSDP/ZeRO 需要 AllGather 和 ReduceScatter；tensor parallel 会在层内频繁通信。通信越频繁，拓扑越重要。

24.5 为什么拓扑会进入调度

调度器如果只知道“还剩 8 张 GPU”，可能把通信密集任务放到低效拓扑里。平台需要知道：

  • GPU 是否在同一节点。
  • GPU 之间是否有 NVLink/NVSwitch。
  • GPU 和网卡的 NUMA/PCIe 关系。
  • 跨节点是否有 RDMA。
  • 当前任务是数据并行、张量并行还是流水线并行。

这就是后面拓扑感知调度的硬件基础。

24.6 一个拓扑案例

同样申请 8 张 GPU：

放置方式 适合任务 不适合任务
单机 8 GPU，NVLink/NVSwitch tensor parallel、大模型推理、通信密集训练 资源紧张时可能排队久
两台机器各 4 GPU，RDMA DDP、部分 FSDP 层内通信频繁的 tensor parallel
八台机器各 1 GPU，以太网 松耦合批任务、低通信推理副本 通信密集分布式训练

调度器如果只看 GPU 数量，三种放置都可能满足请求；但对性能来说，它们不是同一种资源。

24.7 平台要暴露哪些拓扑信息

后面进入 K8s 和调度时，至少要能表达：

  • GPU 型号和显存容量。
  • GPU 间 NVLink/NVSwitch 拓扑。
  • GPU 到 NIC 的距离。
  • CPU NUMA 与 GPU/网卡关系。
  • 节点是否支持 RDMA。
  • MIG/vGPU 等切分方式。

这些信息最终会影响 node selection、pod placement、gang scheduling、拓扑感知和队列策略。

24.8 本章小结

  • 多 GPU 性能不是线性扩展。
  • 通信库、互联路径和拓扑会决定扩展效率。
  • 拓扑信息必须进入平台和调度系统，否则 GPU 数量满足也可能性能很差。
《NVIDIA NCCL User Guide》. 不详. https://docs.nvidia.com/deeplearning/nccl/user-guide/docs/.
