10  优化器与参数更新

10.1 本章要回答的问题

backward 得到 gradient 后，模型还没有变好。真正修改 parameters 的，是 optimizer。

optimizer 的本质，是根据 gradient 和自身保存的状态，决定每个 parameter 应该移动多少。

optimizer 不是按钮，而是参数更新账房

分片ZeRO/FSDP 会切分这些状态

只保存 weights 通常适合推理；要继续训练，必须把 optimizer、scheduler、step 和随机状态一起当作训练状态。

10.2 从 SGD 到 Adam

最简单的更新可以写成：

parameter = parameter - learning_rate * gradient

这就是 SGD 的基本直觉。Adam 这类 optimizer 会额外保存动量、方差等状态，让更新更稳定，但代价是显存占用更大。

optimizer 状态 含义 系统影响
learning rate 每次更新步长 影响收敛和稳定性
momentum 平滑更新方向 增加状态
variance 估计梯度波动 增加状态
master weights 混合精度下可能保存高精度副本 增加显存

PyTorch 的 torch.optim 文档把 optimizer 设计成接收 parameters 并更新它们的对象 (《torch.optim》, 不详)。对算法开发者来说，这是一个 API；对平台工程师来说，它意味着训练进程里除了 weights 和 gradient，还有一批必须被管理的 optimizer state。

10.3 为什么 optimizer 是系统问题

optimizer 看起来是算法组件，但它直接影响系统：

  • optimizer state 会占显存。
  • ZeRO、FSDP 会切分 optimizer state。
  • checkpoint 必须保存 optimizer state，否则 resume 后训练轨迹会变。
  • learning rate schedule 需要和 step 计数一致。

因此，平台不能只保存模型 weights，还要保存完整训练状态。

10.4 一个显存直觉：Adam 为什么贵

假设模型有 70 亿个 parameters，使用 BF16 weights 时，weights 本身约 14GB。但训练不是只放 weights：

内容 粗略直觉
weights 当前模型参数
gradient 每个 parameter 的更新方向
optimizer state Adam 的一阶动量、二阶动量等状态
master weights 某些混合精度策略下保留的高精度副本

所以训练显存可能远大于推理显存。ZeRO、FSDP、optimizer state offload 这些技术，本质上就是在问：这些 optimizer state 是否一定要每张 GPU 都完整保存？

10.5 learning rate schedule 是训练状态的一部分

optimizer 不只看当前 gradient，还会受到 learning rate schedule 影响。常见 schedule 包括 warmup、cosine decay、step decay 等。它们的共同点是：当前 step 数会影响下一次参数更新。

这会带来一个平台问题：resume 时如果只恢复 weights，不恢复 optimizer state、scheduler state 和 step，训练表面上能继续跑，但轨迹已经变了。

状态 不恢复会怎样
optimizer state momentum/variance 丢失，更新方向突变
scheduler state learning rate 回到错误阶段
step / epoch 日志、评估、checkpoint 和数据游标错位
mixed precision scaler 可能导致数值稳定性变化

所以 checkpoint 不是保存一个模型文件，而是保存一次训练实验的可恢复状态。

10.6 optimizer 与分布式切分

大模型训练里，optimizer state 经常比想象中更贵。ZeRO/FSDP 这类技术会把 parameters、gradient、optimizer state 在多个 worker 之间切分，减少每张 GPU 的重复存储。

这种优化改变了平台假设：

  • checkpoint 可能是 sharded checkpoint，不再是一个简单文件。
  • worker 数量和并行策略会影响状态切分方式。
  • 恢复时需要知道原来的并行配置，或者能做重分片。
  • 调度器不能随意改变分布式训练拓扑，否则恢复和性能都会受影响。

10.7 本章小结

optimizer 是 parameters 更新的执行者。训练平台要把 optimizer state 当作一等状态管理，因为它影响显存、checkpoint 和恢复一致性。

《torch.optim》. 不详. https://docs.pytorch.org/docs/stable/optim.html.
