9  前向计算、loss 与反向传播

9.1 本章要回答的问题

一个 batch 进入模型后，训练 step 的核心链路是：

batch -> forward -> loss -> backward -> gradient

PyTorch autograd 教程把自动微分描述为神经网络训练的核心能力：forward 产生输出，loss 衡量误差，autograd 根据计算图自动计算各 parameters 的 gradient (《A Gentle Introduction to torch.autograd》, 不详)。

Excalidraw Style

autograd 的账本：forward 留痕，backward 追账

手绘风格源文件：fig/excalidraw/autograd-loop.excalidraw

Batch 输入 tensor shape、dtype、device 决定后续计算量和显存压力。
Forward 计算 output 每一层产生 activation；训练时这些中间值要被保留。
Loss 错得有多远 把 output 和 target 的差距变成一个可优化标量。
Backward 产生 gradient 沿计算图反向传播，计算每个 parameter 应该怎么改。
activation不是“临时垃圾”。backward 需要它，所以长序列和大 batch 会很吃显存。
gradient不是新参数，而是更新方向。分布式训练通常要同步它。
checkpointing可以少存 activation、需要时重算，用计算换显存。

9.2 forward 是什么

forward 是模型根据当前 parameters 对 input 进行计算。对分类模型，它输出每个类别的分数；对语言模型，它输出下一个 token 的分布。

forward 会产生 activation。这些 activation 不是临时垃圾，训练时 backward 需要用它们计算 gradient。因此，模型越深、batch 越大、sequence length 越长，activation 显存越大。

9.3 loss 是什么

loss 是“模型错得有多远”的数值化表达。训练不是直接追求输出好看，而是最小化 loss。

常见例子：

  • 分类任务用 cross entropy 衡量预测类别和真实类别的差距。
  • 语言模型用下一个 token 的 cross entropy 训练。
  • 回归任务可能用 MSE 衡量预测值和目标值距离。

loss 的选择属于算法问题，但平台要知道它决定了 backward 是否能正常产生 gradient，以及训练指标如何解释。

9.4 backward 是什么

backward 根据 loss 反向计算每个 parameter 的 gradient。gradient 不是“更新后的参数”，而是“参数应该往哪个方向调整”的信号。

分布式训练中，多个 worker 会各自算出 gradient，然后通过通信同步或聚合。这也是为什么训练会对网络和拓扑敏感。

PyTorch 的 DistributedDataParallel 会在多个进程之间同步 gradient，使多个副本保持一致更新 (《DistributedDataParallel》, 不详)。从系统角度看，DDP 不是“多启动几个进程”，而是把 backward 之后的 gradient 同步变成训练 step 的一部分。

9.5 autograd 带来的系统含义

autograd 让算法开发者不必手写每个 operator 的求导公式，但它会带来两个系统事实：

  • forward 期间必须保留 backward 需要的 activation。
  • 计算图越深、batch 越大、sequence length 越长，activation 显存越高。

这就是 activation checkpointing、gradient accumulation、mixed precision、ZeRO/FSDP 等优化出现的背景。它们不是孤立技巧，而是在“显存、计算、通信”之间重新做权衡。

PyTorch activation checkpointing 文档对这个权衡讲得很明确：checkpointing 会省掉部分 forward activation 的保存，在 backward 时重新计算这些 activation，以计算开销换显存 (《torch.utils.checkpoint》, 不详)。这句话对平台工程非常关键，因为它解释了一个常见现象：同一个模型开启 checkpointing 后，显存下降了，但 step time 可能变长。

9.6 把 autograd 看成运行时账本

可以把训练 step 想成一套账本系统：

动作 账本里新增什么 系统代价
forward output、activation、计算图依赖 占用显存，决定 backward 能不能追溯
loss 一个可优化的标量目标 指标解释和梯度入口
backward 每个 parameter 的 gradient 计算开销、显存、分布式同步
gradient accumulation 多个 micro-batch 的 gradient 累积 用时间换更大的有效 batch
activation checkpointing 少存一部分 activation 用重算换显存

这样理解后，很多训练优化就不再是碎片技巧。它们都在改同一张账本：哪些中间结果要存，哪些可以重算，哪些 gradient 需要立刻同步，哪些可以累积后再更新。

9.7 micro-batch、global batch 和 gradient accumulation

训练里经常同时出现三个 batch 概念：

名词 含义 为什么重要
micro-batch 单次 forward/backward 放进一张 GPU 的样本数 受单卡显存限制
per-device batch 每个 worker 一次处理的样本数 决定每个进程的计算和显存
global batch 所有 worker、所有累积步合起来的有效 batch 影响收敛和学习率设置

如果显存放不下大 batch，可以用 gradient accumulation：多个 micro-batch 依次 forward/backward，把 gradient 累积起来，最后再 optimizer step。它的本质是用更多时间换更大的有效 batch。

global batch = per-device batch x worker 数 x accumulation steps

这个公式很重要。后面讲调度和资源扩缩容时，worker 数改变不只是资源改变，也会改变 global batch。如果学习率、scheduler、日志和 checkpoint 没有一起调整，训练语义就会变化。

9.8 三个常见误区

误区 为什么错
forward 结束后 activation 就没用了 训练 backward 需要它们，所以它们是显存大头之一
gradient 越快同步越好 同步频率、通信重叠和 bucket 大小会影响整体 step time
显存不够只要减 batch 还可以用 activation checkpointing、混合精度、gradient accumulation、参数/优化器状态切分

9.9 本章小结

  • forward 产生 output 和 activation。
  • loss 把 output 和 target 的差距变成一个可优化目标。
  • backward 根据 loss 计算 gradient。
  • activation 和 gradient 是训练显存的重要来源。
